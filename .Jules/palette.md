## 2024-06-07 - Add Keyboard Focus States to Floating Modals
**Learning:** In the `campus-app` application, custom dropdown components like the `NotificationBell.tsx` popup that rely heavily on `framer-motion` for their visibility state often omit standard keyboard focus rings (`focus-visible:ring-1`) for embedded icon-only action buttons (e.g., mark all read, close). This creates a critical accessibility issue where screen reader and keyboard-only users cannot perceive which secondary action is focused when interacting with the modal.
**Action:** When working with or creating custom interactive popups/modals in this application, explicitly add `focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-ring` to all embedded icon-only utility buttons to ensure keyboard interaction is visually distinct, and ensure they have accompanying `aria-label` attributes corresponding to the component's localization language (e.g., English for NotificationBell).
