# PHASE 2 — WRITE PATH HARDENING

All write operations must follow this exact pattern:

```ts
setLocalLoading(true)
try {
  await firestoreWrite()
  showSuccess()
} catch (error) {
  handleError(error)
} finally {
  setLocalLoading(false)
}
```

Requirements:

* No write operation may depend on listener confirmation
* No UI state may wait for DataEngine processing
* No global loading lock may block the entire app
* Every async action must always resolve or fail gracefully

Validate all:

* location creation
* post creation
* post editing
* location editing
* deletion flows
* profile updates
